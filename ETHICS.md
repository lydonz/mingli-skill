# 使用边界与隐私说明

## 项目定位

MingLi Skill 用于传统命理文化研究、历法计算、自我认知和 Agent 工具开发。它输出的是基于规则与符号体系的解释，不是对个人命运、健康、财富或关系的事实判断。

## 不确定性

- 出生时间误差、出生地点、历法年界和流派规则都会影响排盘或解释。
- 四柱与节气可以按历法后端回归验证；命理解读不能据此宣称能准确预测未来事件。
- 输出应使用“倾向”“可能”“需要结合现实信息”等表述，不能把命理语言包装成确定结论。
- 已知经历可以帮助用户复盘，但不能用于反推规则或证明未来断语。

## 高风险事项

不得用本项目的结果替代：

- 医疗诊断、用药、心理危机干预或紧急求助。
- 法律意见、诉讼策略或公共安全判断。
- 投资、借贷、保险、赌博或其他金融决策。
- 雇佣、录取、解雇、信贷、住房或任何可能造成歧视的决定。
- 婚姻、离婚、怀孕、迁移、辞职等重大人生决定。

涉及上述问题时，应建议咨询合格的专业人士，并把命理内容限制为文化参考。

## 隐私与同意

出生日期、时间、地点、性别、命盘和报告均可能构成个人敏感信息。

- 为他人排盘前，应取得对方同意。
- 不要将完整命盘、姓名、出生地点或生成的报告公开发布。
- 公开示例应匿名化，或使用已经公开的历史资料。
- HTML 报告仅在调用方显式指定 `output_path` 时写入本地文件；项目不会主动上传出生信息或报告内容。
- 使用城市解析和真太阳时时，应告知用户地点输入的用途。
- 本地知识包检索只读取随项目分发或由调用方本地安装的 Markdown 文件；不得把出生资料、命盘、问题文本或检索结果发送到外部检索、嵌入或分析服务。
- 知识包引用必须保留来源、许可证、固定版本和行级定位；它们是传统文化资料，不得用作现实预测或个人属性判断的证据。
- 解读文本必须与命盘的 `chart_id` 和结构化证据绑定，并标明不确定性；不得把用户反馈称作“校准”或据此宣称更高预测准确率。
- 不得将未转义的用户输入、LLM 输出或解释文本插入 HTML 报告。
- 规则选项建议不能用于健康、伤害、法律或涉及死亡、医疗、投资、借贷、赌博、破产、怀孕、离婚等高风险主题；必须返回可见的抑制状态。

## Agent 输出原则

使用本项目的 Agent 应：

1. 先说明输入条件、时间口径和数据质量状态。
2. 区分历法事实、规则推演与现实建议。
3. 解释模型冲突、缺失输入和候选命盘，而不是隐藏它们。
4. 避免恐吓、羞辱、歧视、操纵或宿命论语言。
5. 在用户表现出严重焦虑、绝望或自伤风险时，优先鼓励寻求现实支持和专业帮助。
